# data-fire 小白使用指南

这份指南写给第一次使用 data-fire 的 Scratcher。你不需要懂后端、数据库、构建命令，只需要知道：在作品里放好数据积木，玩家运行作品后，数据就会进入看板，你再打开看板查看分析结果。

---

## 1. data-fire 是用来做什么的

如果你做了一个 Scratch 游戏，普通的播放量只能告诉你“有人玩过”，但不能告诉你这些问题：

- 玩家玩了多久？
- 有多少玩家第二次回来玩？
- 玩家卡在哪一关？
- 哪个按钮、道具、玩法被触发最多？
- 最高分、平均分是多少？
- 新手教程有多少人完成到最后一步？

data-fire 就是帮你记录这些数据的 Scratch 拓展。

它分成两部分：

1. **Scratch 作品里的拓展积木**：你把积木放进作品，积木负责记录玩家行为。
2. **Dashboard 数据看板**：你打开看板地址，就能看到图表和统计结果。

整条流程可以理解成：

```text
玩家玩你的作品 → data-fire 积木记录数据 → 数据自动上传 → 你打开 Dashboard 看图表
```

---

## 2. 第一次使用时先做什么

### 第一步：在作品里添加 data-fire 拓展

在支持自定义 Scratch 拓展的平台里打开你的作品，然后添加 data-fire 拓展。

添加成功后，积木区会出现一个新的分类，名字通常是：

```text
玩家数据分析
```

如果你没有看到这个分类，说明拓展还没有加载成功，需要先确认当前平台是否已经接入 data-fire，或者向项目维护者获取可用的拓展入口。

### 第二步：先放入“开启自动数据收集”积木

在作品开始运行的位置放入：

```text
开启自动数据收集
```

最推荐的放法是放在绿旗下面：

```text
当绿旗被点击
开启自动数据收集
```

这样玩家每次点绿旗开始玩时，data-fire 就会自动记录：

- 本次游玩什么时候开始；
- 本次游玩什么时候结束；
- 本次玩了多久；
- 这个玩家是不是回头玩家；
- 玩家关闭页面时的兜底结束记录。

只想先体验最基础的数据分析时，放这一个积木就够了。

## 3. 怎么拿到自己的看板地址

拓展里有一个报告积木：

```text
看板地址
```

它会返回当前作品对应的数据看板链接，格式类似：

```text
http://localhost:5173/p/你的作品ID/
```

你可以这样使用它：

```text
当绿旗被点击
开启自动数据收集
说 看板地址
```

运行作品后，角色说出的那串链接就是你的数据看板地址。

你也可以把“看板地址”的结果保存到变量里，方便自己复制：

```text
当绿旗被点击
开启自动数据收集
将 [我的看板地址 v] 设为 看板地址
```

以后你想看这个作品的数据，就打开这个地址。

## 4. 自动数据会记录什么

只要你使用了：

```text
开启自动数据收集
```

系统就会自动记录一些通用数据。

| 数据       | 它表示什么                       | 你能用它判断什么                 |
| ---------- | -------------------------------- | -------------------------------- |
| 会话数     | 玩家一共开始了多少次游玩         | 作品有没有被频繁打开             |
| 唯一玩家数 | 大概有多少个不同玩家玩过         | 玩家规模有多大                   |
| 平均时长   | 玩家平均玩多久                   | 游戏是不是太短、太难或不够吸引人 |
| 完整率     | 玩家是否自然结束，而不是中途关闭 | 玩家是否容易中途退出             |
| 回头率     | 有多少玩家再次回来玩             | 作品是否有复玩价值               |

这些数据不需要你额外埋点，适合所有作品。

## 5. 想记录更具体的行为怎么办

自动数据只能告诉你“玩家玩了多久、有没有回来”。

如果你想知道“玩家做了什么”，就要在对应的游戏逻辑里主动放自定义记录积木。

### 5.1 记录普通事件

积木：

```text
记录事件 [name]
```

适合记录某个动作发生过，比如：

```text
记录事件 [点击开始按钮]
记录事件 [获得金币]
记录事件 [死亡]
记录事件 [通关第一关]
```

建议事件名写成人能看懂的中文或简单英文。只要前后一致，看板就能统计。

例如你想知道玩家有没有点“开始游戏”按钮，可以把积木放在按钮角色的点击逻辑里：

```text
当角色被点击
记录事件 [点击开始按钮]
广播 [开始游戏 v]
```

### 5.2 记录带数字的事件

积木：

```text
记录事件 [name] 值为 [value]
```

适合记录“事件 + 数字”，比如：

```text
记录事件 [到达关卡] 值为 [3]
记录事件 [购买道具] 值为 [50]
记录事件 [获得金币] 值为 [10]
```

如果你的游戏有很多关卡，可以在进入关卡时记录：

```text
记录事件 [到达关卡] 值为 当前关卡
```

这样之后就能看出玩家大多到第几关。

### 5.3 记录带文字详情的事件

积木：

```text
记录事件 [name] 详情 [detail]
```

适合记录“事件 + 一段文字”，比如：

```text
记录事件 [选择角色] 详情 [战士]
记录事件 [失败原因] 详情 [被怪物碰到]
记录事件 [选择难度] 详情 [困难]
```

如果你想知道玩家最常选哪个角色，就可以在选角色时记录角色名。

## 6. 指标、计数器、分数和漏斗怎么用

### 6.1 指标：记录一个会变化的数值

积木：

```text
记录指标 [name] 当前值 [value]
```

指标适合记录某个随时间变化的数字，例如：

```text
记录指标 [血量] 当前值 血量
记录指标 [金币数] 当前值 金币
记录指标 [速度] 当前值 速度
```

Dashboard 里可以输入指标名查看折线图。

如果你记录的是：

```text
记录指标 [金币数] 当前值 金币
```

那你在 Dashboard 的“指标趋势”里就输入：

```text
金币数
```

### 6.2 计数器：记录累计次数或当前累计值

计数器有两个积木：

```text
计数器 [name] 增加 [delta]
计数器 [name] 设为 [value]
```

常见用法：

```text
计数器 [死亡次数] 增加 [1]
计数器 [拾取金币数] 增加 [1]
计数器 [当前连击数] 设为 连击数
```

“增加”适合每次发生就加一点。

“设为”适合直接把当前变量值同步过去。

Dashboard 的“计数器当前值”会显示每个计数器最新是多少。

### 6.3 分数：记录最终成绩

积木：

```text
提交分数 [value]
```

适合在一局游戏结束时使用：

```text
当游戏结束
提交分数 分数
```

Dashboard 的“分数分布”会显示：

- 提交过多少次分数；
- 最高分；
- 最低分；
- 平均分；
- 分数 Top 排行。

### 6.4 漏斗：记录一个流程走到哪一步

积木：

```text
漏斗 [funnel] 进入步骤 [step]
```

漏斗适合分析“一个流程里，玩家在哪一步流失”。

比如新手教程有 4 步：

```text
漏斗 [新手教程] 进入步骤 [开始教程]
漏斗 [新手教程] 进入步骤 [学会移动]
漏斗 [新手教程] 进入步骤 [学会攻击]
漏斗 [新手教程] 进入步骤 [完成教程]
```

Dashboard 的“漏斗转化”里输入：

```text
新手教程
```

就能看到每一步有多少玩家到达，以及从上一步到下一步的转化率。

## 7. Dashboard 怎么看

打开“看板地址”后，你会进入对应作品的数据看板。

页面顶部会显示：

```text
data-fire 数据看板
作品 ID：你的作品ID
```

右上角有“刷新”按钮。如果你刚刚运行完作品但数据还没出现，可以等几秒后点一次刷新。

### 7.1 总览数字怎么看

顶部第一排是最重要的概览：

| 卡片       | 怎么理解               |
| ---------- | ---------------------- |
| 总会话数   | 作品被开始游玩的次数   |
| 唯一玩家数 | 大概有多少不同玩家玩过 |
| 平均时长   | 每次游玩平均持续多久   |
| 完整率     | 有多少会话是自然结束的 |
| 回头率     | 有多少玩家回来玩过多次 |

如果平均时长很短，可能说明开头不够吸引人、难度太高，或者玩家不知道怎么玩。

如果回头率高，说明玩家愿意再次回来玩。

### 7.2 玩家回头率怎么看

“玩家回头率”卡片会显示：

- 唯一玩家；
- 新玩家；
- 回头玩家；
- 人均访问次数。

如果回头玩家很多，说明作品有复玩价值。

如果新玩家很多但回头玩家很少，说明作品可能第一眼不错，但留不住人。

### 7.3 会话时长分布怎么看

“会话时长分布”会把玩家的游玩时长分成几个区间。

你可以用它判断玩家是否大量在开头流失。

例如很多玩家只玩了几秒就离开，通常说明：

- 开始界面不清楚；
- 操作说明不明显；
- 第一关太难；
- 加载或等待时间太长。

### 7.4 事件趋势怎么看

“事件趋势”可以看事件每天发生了多少次。

默认是“全部事件”。

如果你想看某一个事件，就切换到“单事件”，输入你在积木里写过的事件名，比如：

```text
点击开始按钮
死亡
通关第一关
```

然后点击“查询”。

事件名必须和你在 Scratch 积木里写的一样，否则查不到对应数据。

### 7.5 热门事件 Top 怎么看

“热门事件 Top”会显示触发次数最多的事件。

如果“死亡”特别高，可能说明游戏偏难。

如果“点击开始按钮”很高但“通关第一关”很低，可能说明玩家开始玩了，但第一关流失严重。

### 7.6 指标趋势怎么看

“指标趋势”需要你输入指标名。

比如你在作品里用了：

```text
记录指标 [金币数] 当前值 金币
```

那就在 Dashboard 里输入：

```text
金币数
```

然后点击“查询”。

它会画出这个指标随时间变化的折线。

### 7.7 计数器当前值怎么看

“计数器当前值”会列出所有计数器的最新值。

比如：

- 死亡次数；
- 拾取金币数；
- 使用技能次数；
- 打败怪物数。

如果这里显示“暂无计数器”，说明你的作品还没有触发过计数器积木。

### 7.8 分数分布怎么看

“分数分布”会显示玩家提交的分数统计。

你需要先在作品里使用：

```text
提交分数 分数
```

如果没有提交过分数，Dashboard 就不会有分数统计。

### 7.9 漏斗转化怎么看

“漏斗转化”需要输入漏斗名。

比如你在作品里写的是：

```text
漏斗 [新手教程] 进入步骤 [开始教程]
```

那 Dashboard 里就输入：

```text
新手教程
```

然后点击“查询”。

漏斗图适合看教程、关卡流程、购买流程等分步骤行为。

## 8. 推荐的新手埋点方案

如果你不知道该记录什么，可以先照着下面这个最小方案放。

### 游戏开始时

```text
当绿旗被点击
开启自动数据收集
记录事件 [游戏开始]
```

### 玩家点击开始按钮时

```text
当开始按钮被点击
记录事件 [点击开始按钮]
```

### 进入关卡时

```text
记录事件 [到达关卡] 值为 当前关卡
漏斗 [主线关卡] 进入步骤 当前关卡
```

### 玩家死亡时

```text
记录事件 [死亡]
计数器 [死亡次数] 增加 [1]
```

### 玩家获得金币时

```text
记录事件 [获得金币] 值为 获得数量
计数器 [金币总数] 增加 获得数量
记录指标 [当前金币] 当前值 金币
```

### 游戏结束时

```text
提交分数 分数
记录事件 [游戏结束]
```

这个方案已经能让你看到：

- 有多少人开始游戏；
- 有多少人点击开始；
- 玩家大多到第几关；
- 死亡次数多不多；
- 金币获得情况；
- 分数高低；
- 玩家整体留存情况。

## 9. 命名建议：事件名要稳定

data-fire 是按你写的名字统计数据的，所以名字要尽量固定。

推荐：

```text
死亡
点击开始按钮
通关第一关
金币数
新手教程
```

不推荐同一个意思写很多种：

```text
死亡
玩家死亡
死了
game over
```

因为 Dashboard 会把它们当成 4 个不同事件。

如果你想长期看数据，建议先定好名字，以后不要频繁改。

## 10. 为什么我刚打开看板没有数据

常见原因有这些：

### 1. 作品还没有真正运行过

只打开编辑器但没有点绿旗，通常不会产生完整数据。

请点绿旗运行一次作品。

### 2. 没有放“开启自动数据收集”

如果没有这个积木，自动会话、时长、回头率不会开始记录。

请把它放到绿旗下面。

### 3. 数据上传需要几秒

data-fire 会批量上传数据，不一定每触发一次积木就马上出现在看板。

请等待几秒，再点击 Dashboard 右上角“刷新”。

### 4. 查询的名字和积木里的名字不一致

比如积木里写的是：

```text
金币数
```

但 Dashboard 里输入：

```text
金币
```

这两个名字不同，所以查不到。

请复制完全一致的名字。

### 5. 这个作品还没有触发过对应积木

例如你没有用过“提交分数”，分数分布就不会有数据。

你没有用过“漏斗”积木，漏斗图就不会有数据。

## 11. 使用时要注意什么

### 不要记录隐私信息

不要把玩家的真实姓名、手机号、密码、聊天内容等隐私内容写进事件详情。

data-fire 适合记录游戏行为，不适合记录私人信息。

### 不要每一帧都记录大量事件

不要在永远循环里每一帧都放大量记录积木，例如：

```text
重复执行
记录事件 [正在移动]
```

这样会产生太多无用数据。

如果你确实想记录变化中的数值，可以隔一段时间记录一次，或者只在关键变化时记录。

### 事件名改了以后，旧数据不会自动合并

比如你一开始叫：

```text
死亡
```

后来改成：

```text
玩家死亡
```

Dashboard 会把它们当成两个事件。

---

## 12. 一句话总结

最简单的用法：

```text
当绿旗被点击
开启自动数据收集
```

想看数据：

```text
运行作品 → 使用“看板地址”拿到链接 → 打开 Dashboard → 等几秒后刷新
```

想看更具体的游戏行为：

```text
在按钮、关卡、死亡、得分、教程步骤等关键位置放自定义记录积木
```

只要记住这三件事，你就已经会用 data-fire 做基础玩家数据分析了。
